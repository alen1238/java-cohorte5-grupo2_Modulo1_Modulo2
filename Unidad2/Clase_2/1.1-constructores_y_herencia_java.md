# Constructores en clases padre e hija (Java)

## Caso 1: El padre tiene varios constructores (vacío y con parámetros)

La clase hija no tiene que escribir `super()` si quiere usar el
constructor vacío.

``` java
class Padre {
    public Padre() {
        System.out.println("Padre: constructor vacío");
    }

    public Padre(int x) {
        System.out.println("Padre: constructor con parámetro " + x);
    }
}

class Hija extends Padre {
    public Hija() {
        System.out.println("Hija: constructor");
    }
}
```

### ¿Qué constructor llama la hija?

Java inserta esto automáticamente:

``` java
super();  // Llama al constructor vacío
```

**Resultado al crear:**

``` java
new Hija();
```

**Salida:**

    Padre: constructor vacío
    Hija: constructor

------------------------------------------------------------------------

## Caso 2: La hija quiere llamar un constructor específico del padre

Aquí **SÍ** debes escribir `super(...)`.

``` java
class Hija extends Padre {
    public Hija() {
        super(50);   // Llamando al constructor con parámetro
        System.out.println("Hija: constructor");
    }
}
```

**Salida:**

    Padre: constructor con parámetro 50
    Hija: constructor

------------------------------------------------------------------------

## Caso 3: El padre **NO** tiene constructor vacío

Esto obliga a la hija a llamar explícitamente a un constructor del
padre.

``` java
class Padre {
    public Padre(int x) {
        System.out.println("Padre: " + x);
    }
}
```

La hija intenta esto:

``` java
class Hija extends Padre {
    public Hija() { }
}
```

**ERROR**

Java intenta hacer esto internamente:

``` java
super(); // pero NO existe
```

### Solución correcta:

``` java
class Hija extends Padre {
    public Hija() {
        super(10); // Obligatorio
        System.out.println("Hija: constructor");
    }
}
```

------------------------------------------------------------------------

##  Regla de oro para recorda y enseñar

-   Si el padre **tiene constructor vacío**, el hijo **NO** está
    obligado a escribir `super()`.
-   Si el padre **NO tiene constructor vacío**, el hijo **SÍ** está
    obligado a llamar explícitamente a uno de los constructores del
    padre.
